Elementos Semânticos

Segundo o dicionário, semântica é o ramo da linguística que estuda o significado das palavras, portanto elemento semântico é aquele que dá significado às informações inseridas na página html, esse significado não é passado ao usuário e sim aos motores de busca e indexação da Internet.

Ao se escrever uma página html, o nosso objetivo é passar ao usuário o conteúdo, os estilos e as interações, mas devemos também passar a semântica dos elementos html para os motores de busca.

Uma página básica, normalmente, tem um cabeçalho, na parte de cima, com o logotipo ou nome da página e um menu de navegação, esse cabeçalho deve ser incluído entre as tags header (não confundir com a tag head).

Abaixo do cabeçalho vem o conteúdo principal da página, com todas as informações relevantes, esse conteúdo principal deve ser incluído entre as tags main.

Abaixo do conteúdo principal da página, normalmente vem um rodapé, com informações de endereço, telefone, copyright, mídias sociais e outros, esse rodapé deve ser incluído entre as tags footer.

O código html ficaria assim:

<!DOCTYPE html>
<html lang=”pt-br”>
    <head>
        <meta charset=”utf-8”>
        <title>Título da Página</title>
    </head>
    <body>
    	<header>
            <!-- Incluir o cabeçalho aqui. -->
    	</header>
    	<main>
            <!-- Incluir o conteúdo principal aqui. -->
    	</main>
        <footer>
            <!-- Incluir o rodapé aqui. -->
    	</footer>
    </body>
</html>

Note as linhas que começam com <!-- e terminam com --> são comentários e não serão lidos ou executados pelo navegador. Pressionando as teclas Ctrl + Shift + / o Sublime Text já escreve os caracteres de comentário automaticamente, pressionando as teclas Ctrl + / o Sublime Text transforma toda a linha onde está o cursor em comentário.

Os comentários podem ser inline, em uma única linha, como nos exemplos acima, podem ser inline code, escritas ao lado do código ou podem ser multiline ou de múltiplas linhas, onde na primeira linha é escrito <!-- e na última linha é escrito --> e todas as linhas que estiverem dentro dessas tags será um comentário.

Continuando com os elementos semânticos, dentro do header podemos utilizar o elemento nav que define um menu de navegação do site, o nav também pode ser usado no elemento aside (que veremos mais tarde), no elemento footer e em outros elementos, mas o seu uso é só para links de navegação do site, se for links de paginação, links externos, como o de mídias sociais, o nav não deve ser utilizado.

Dentro do main podemos utilizar outros elementos semânticos, dentre outros, o section, que define uma seção da página e o article, que define um artigo da página. O conteúdo do article deve ter um sentido completo, podendo ser reutilizado no mesmo site, em outro site, em um fórum ou blog. Tanto o section como o article deve ter pelo menos um heading.

Headings devem ser usados para representar semanticamente um título na página, pode ser utilizado o elemento h1, h2, h3, h4, h5 ou h6, dependendo do seu nível de importância e relevância, sendo o h1 o mais importante e o h6 o menos importante, lembrando que cada página html só deve ter um elemento h1, os elementos h2 a h6 podem se repetir na página, mas não se deve passar de h2 para h4, por exemplo.

Heading é um elemento semântico de texto, portanto só deve ser utilizado em texto, vemos na Internet o heading sendo usado erroneamente, por exemplo, no logotipo do header da página html.

O heading é muito importante para os motores de busca e também são renderizados de forma diferente, por padrão, pelo navegador, sendo o h1 o elemento com a maior fonte e o h6 o elemento com a menor fonte, mas o tamanho da fonte não é importante, até mesmo porque podemos, e devemos, estilizar o tamanho da fonte, o tipo da fonte, a cor da fonte e outros atributos através do CSS.

Veja o código para exibir as tags h1 a h6.

<!DOCTYPE html>
<html>
    <head>
        <title>Título da Página</title>
    </head>
    <body>
    	<h1>Testando h1</h1>
    	<h2>Testando h2</h2>
    	<h3>Testando h3</h3>
    	<h4>Testando h4</h4>
    	<h5>Testando h5</h5>
    	<h6>Testando h6</h6>
    </body>
</html>

Veja as tags h1 a h6 renderizadas, por padrão, pelo navegador, sem uso do CSS.

Lembre-se que as headings devem ser usadas em títulos, para o texto normal usamos o elemento p de parágrafo, veja um exemplo prático do uso de headings e parágrafos.

Código:

<!DOCTYPE html>
<html>
    <head>
        <title>Título da Página</title>
    </head>
    <body>
    	<h1>Receitas da Vovó</h1>
    	<h2>Receitas de Bolo</h2>
    	<h3>Bolo de Chocolate</h3>
    	<h4>Ingredientes:</h4>
        <p>
            Lista de Ingredientes do Bolo de Chocolate
        </p>
    	<h4>Modo de Preparo:</h4>
        <p>
            Lista de Preparo do Bolo de Chocolate
        </p>
        <h3>Bolo de Fubá</h3>
        <h4>Ingredientes:</h4>
        <p>
            Lista de Ingredientes do Bolo de Fubá
        </p>
        <h4>Modo de Preparo:</h4>
        <p>
            Lista de Preparo do Bolo de Fubá
        </p>
    </body>
</html>

Como será mostrado no navegador:

Note que no exemplo acima, só é usado um h1, pois cada página só deve ter um título principal, o h2 é usado para o tipo da receita, nesse caso, Receitas de Bolo, o h3 é usado para o tipo de bolo, nesse caso Bolo de Chocolate e Bolo de Fubá, o h4 é usado para definir os Ingredientes e o Modo de Preparo, note que mesmo após o h4, usamos o h3 novamente, para definir o tipo de bolo, o parágrafo é usado para escrever a Lista de Ingredientes e o Modo de Preparo, mais a frente veremos que seria melhor, mais certo e mais semântico se usarmos o elemento ul e ol para definir listas, nesse exemplo, uma ul, unordered list, lista não ordenada para os Ingredientes e uma ol, ordered list, lista ordenada para o Modo de Preparo.

Com relação ao posicionamento do h1 ao h6, não se esqueça que o importante não é a ordem em que elas são escritas na página e sim a sua relevância, no exemplo acima, se no final da página adicionarmos Receitas de Doces, devemos utilizar o h2, que foi o heading usado em Receitas de Bolos.
