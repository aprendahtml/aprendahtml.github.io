Iniciando com Box Model

Para entendermos o que é Box Model, primeiro vamos entender o que é um box, ou caixa, ou bloco.

O arquivo html é lido e renderizado pelo navegador de cima pra baixo e da esquerda para a direita, assim como você está lendo essa página e todo elemento html é renderizado como um bloco retangular, esse bloco é colocado, empilhado, um abaixo do outro ou um ao lado do outro, dependendo se o elemento html é em bloco (block) ou em linha (inline).

Os elementos block são empilhados um abaixo do outro e ocupa todo o espaço lateral da página ou o espaço definido pelas propriedades css, width (largura) e height (altura); ou se o espaço não for definido, pelo espaço definido pela sua tag pai.

Os elementos inline são empilhados um ao lado do outro e ocupa o espaço suficiente para exibir o seu conteúdo. Os elementos inline não podem receber algumas propriedades css, por exemplo, width e height.

Exemplos de alguns elementos block:

div, p, h1 a h6, ul, ol, table, form, blockquote.

Exemplos de alguns elementos inline:

span, a, img, td, label, input, select, textarea, b, em.

Exemplo com elementos block:

Código do arquivo exemplo-block.html:

<!DOCTYPE html>
<html lang="pt-br">
<head>
    <link rel="stylesheet" href="exemplo-block.css">
    <meta charset="utf-8">
    <title>Exemplo Block</title>
</head>
<body>
    <div class="bloco-1"></div>
    <div class="bloco-2"></div>
    <div class="bloco-3"></div>
</body>
</html>

Código do arquivo exemplo-block.css:

.bloco-1 {
    width: 200px;
    height: 100px;
    background-color: hsl(352, 100%, 50%);
}

.bloco-2 {
    width: 200px;
    height: 100px;
    background-color: #1815d8;
}

.bloco-3 {
    width: 200px;
    height: 100px;
    background-color: rgb(27, 188, 5);
}

Resultado no navegador:

Note que no arquivo exemplo-block.html usamos o arquivo css externo, exemplo-block.css, através da tag link, com os atributos rel, com o valor stylesheet (folha de estilo) e href, com o valor que é o caminho para o arquivo css.

Usamos também o elemento html div, que é um elemento em bloco genérico, sem nenhum valor semântico, a div é muito usado para fazer divisões na página html, mas nunca use uma div se você puder usar uma tag com valor semântico, como header, nav, main, section, article, aside, footer ou outras.

Outra novidade é o atributo class na tag div, o valor desse atributo pode ser qualquer nome para ser usado como seletor css, por convenção é usado a nomenclatura dash-case. Se usarmos a div como seletor de elemento no css, todas as divs vão herdar as propriedades css do seletor div, mas como nesse exemplo, nós queremos usar uma cor de fundo para cada div, devemos usar um class diferente para cada div. Um elemento html pode ter mais de um class, nesse caso é só dar um espaço depois do nome do primeiro class e escrever o nome do segundo class, tudo dentro das aspas duplas, pode-se usar aspas simples ou até mesmo sem aspas, mas por convenção, se usa aspas duplas. O class pode ser usado em mais de um elemento html, na mesma página ou até em outra página html do site, mas as páginas usadas tem que ter o link para o arquivo css com as propriedades do class.

No lugar do class poderíamos usar o atributo id, que teria o mesmo resultado, a diferença básica entre o class e o id, além de como eles são chamados como elemento no arquivo css, que veremos a seguir, é que o id não pode ser repetido na página html, o id é único e um elemento html só pode ter um id. O id é muito usado para ser referenciado em arquivos JavaScript. Por convenção a nomenclatura do id é lowerCamelCase.

No arquivo exemplo-block.css para usarmos o class como elemento, devemos escrever um ponto (.) antes do nome do class. No caso do id, escrevemos uma cerquilha, ou tralha (#) antes do nome do id.

Vimos, até agora, a definição de cor pelo nome, como red, green, blue e outras, mas nesse arquivo css, fiz a definição de cores com hsl, # (hexadecimal) e rgb, existe ainda as definições com hsla, rgba e gradient, o que veremos mais à frente.

Exemplo com elementos inline:

Código do arquivo exemplo-inline.html:

<!DOCTYPE html>
<html lang="pt-br">
<head>
    <link rel="stylesheet" href="exemplo-inline.css">
    <meta charset="utf-8">
    <title>Exemplo Inline</title>
</head>
<body>
    <span class="inline-1">Inline 1</span>
    <span class="inline-2">Inline 2</span>
    <span class="inline-3">Inline 3</span>
</body>
</html>

Código do arquivo exemplo-inline.css:

.inline-1 {
    background-color: red;
    width: 200px;
    height: 100px;
    font-size: 36px;
}

.inline-2 {
    background-color: green;
    width: 200px;
    height: 100px;
    font-size: 36px;
}

.inline-3 {
    background-color: blue;
    width: 200px;
    height: 100px;
    font-size: 36px;
}

Resultado no navegador:

Usamos também o elemento html span, que é um elemento em linha genérico, sem nenhum valor semântico, o span é muito usado para estilizar uma parte de um texto, mas nunca use um span se você puder usar uma tag com valor semântico.

Note que no arquivo exemplo-inline.css escrevemos, propositalmente, as propriedades width e height, mesmo sabendo que em elementos inline as propriedades width e height não são aplicadas, veja que no resultado do navegador a largura e a altura do elemento inline é definido pelo seu conteúdo, nesse caso pela propriedade font-size com o valor de 36 pixels.

O Box Model é composto de 4 itens: conteúdo (content), padding (preenchimento), border (borda) e margin (margem).

Conteúdo é o que você vai inserir no box, pode ser um título, um parágrafo, uma imagem, um vídeo, um link, o que você quer mostrar para o usuário que vai visualizar o seu site.

Padding é o espaçamento interno entre o box e o conteúdo, padding é sempre transparente.

Border é a borda do box, que pode ser transparente ou ter uma cor definida e pode ter um tamanho definido.

Margin é a margem, o espaçamento externo que o box tem para outro box, margin é sempre transparente.

Vamos fazer um teste no arquivo exemplo-block.html, incluindo um h1 dentro da div com class bloco-2.

Código html:

<!DOCTYPE html>
<html lang="pt-br">
<head>
    <link rel="stylesheet" href="exemplo-block.css">
    <meta charset="utf-8">
    <title>Exemplo Block 2</title>
</head>
<body>
    <div class="bloco-1"></div>
    <div class="bloco-2"><h1>Box Model</h1></div>
    <div class="bloco-3"></div>
</body>
</html>

Resultado no navegador:

Note que o bloco vermelho se afastou, não está mais colado com o bloco azul, isso aconteceu porque o h1 tem por padrão um valor de margin top (acima) e margin bottom (abaixo) e é esse margin top do h1 que afasta o bloco vermelho.

Então podemos dizer, que de certa forma, por padrão, os elementos html utilizam propriedades css automaticamente e nós podemos modificar essas propriedades, algumas pessoas utilizam um arquivo css, normalmente nomeado de reset.css, para resetar, apagar, todas essas propriedades automáticas.

Podemos ver o Box Model das páginas html, com seu padding, border e margin, com os elementos html e suas propriedades css através de Ferramentas do Desenvolvedor do Google Chrome, com o Google Chrome aberto, pressione as teclas Ctrl + Shift + I, por padrão inspecionamos a página através do nome dos elementos na janela que se abriu, para inspecionar os elementos com o cursor do mouse na própria página é só pressionar as teclas Ctrl + Shift + C. Se acostume a usar o Inspetor de Elementos pois é uma ótima ferramenta para entender como a página html é renderizada pelo navegador.
